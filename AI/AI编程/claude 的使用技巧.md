1. 更多并行操作

同时启动 3-5 个 git 工作树，每个工作树都并行运行自己的 Claude 会话。这是最大的生产力提升，也是团队的首选技巧。

2. 每个复杂的计划任务都要从计划模式开始。将你的精力投入到计划中，这样 Claude 就能一次性完成实施。

一个人使用一个 Claude 来制定计划，然后他们再启动第二个 Claude 作为首席工程师来审查它。

另一个人说，一旦出现意外情况，他们就会切换回计划模式并重新计划。不要继续推进。他们还明确告诉 Claude 在验证步骤中进入计划模式，而不仅仅是在构建时。

3. 投资你的 http://CLAUDE.md 。每次纠正后，以：“更新你的 http://CLAUDE.md 以免再次犯同样的错误。” 结尾。Claude 在为自己编写规则方面令人毛骨悚然地擅长。

随着时间的推移无情地编辑你的 http://CLAUDE.md 。不断迭代


4. 创建你自己的技能并将其提交到 git。在所有项目中重复使用。

团队的建议：
- 如果你一天做多次某事，把它变成一个技能或命令
- 创建一个 /techdebt 命令，并在每次会话结束时运行它，以查找和删除重复的代码

5. Claude 会自动修复大多数错误。我们是这样做的：

启用 Slack MCP，然后将 Slack 错误线程粘贴到 Claude 中，只需说“修复”。无需切换上下文。

或者，直接说“去修复失败的 CI 测试”。不要微观管理如何修复。

6. 提升你的提示技巧

a. 挑战 Claude。说“针对这些变更进行严格审查，直到我通过你的测试才提交 PR。”让 Claude 成为你的审查者。或者，说“证明这个能工作”并让 Claude 比较主分支和你的特性分支的行为差异。

b. 在一个平庸的修复后，说：“基于你现在所知的一切，废弃这个，实现优雅的解决方案”

c. 在移交工作前撰写详细的规格并减少歧义。你越具体，输出效果就越好

7. 终端与环境设置

团队非常喜欢 Ghostty！许多人喜欢它的同步渲染、24 位色彩和正确的 Unicode 支持。

为了更方便地管理 Claude，使用/statusline 来定制状态栏，始终显示上下文使用情况和当前的 git 分支。我们中的许多人还会对终端标签进行颜色编码和命名，有时会使用 tmux——每个任务/工作树一个标签。

使用语音输入。你说话的速度是打字的三倍，因此你的提示会变得更加详细。（在 macOS 上按 fn x2）

8. 使用子代理

a. 在任何需要 Claude 投入更多计算资源解决问题的请求中添加"使用子代理"

b. 将单个任务分配给子代理，以保持主代理的上下文窗口干净和专注

c. 通过钩子将权限请求路由到 Opus 4.5——让它扫描攻击并自动批准安全的请求（参见 https://code.claude.com/docs/en/hooks#permissionrequest ）

9. 使用 Claude 进行数据与分析

让 Claude Code 使用"bq" CLI 实时拉取和分析指标。我们已经在代码库中添加了 BigQuery 技能，团队所有人都在 Claude Code 中直接使用它进行分析查询。我个人在 6 个月以上都没有写过一行 SQL。

10. 使用 Claude 学习

团队分享的几个使用 Claude Code 学习的技巧：

a. 在/config 中启用"解释性"或"学习"输出样式，让 Claude 解释其更改背后的原因

b. 让 Claude 生成一个视觉 HTML 演示文稿，解释不熟悉的代码。它制作出令人惊讶的好幻灯片！

c. 请让 Claude 绘制新协议和代码库的 ASCII 图，以帮助你理解它们

d. 建立一个间隔重复学习技能：你解释你的理解，Claude 提出后续问题以填补空白，并存储结果
